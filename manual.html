3URL発行手順

UTM Link Generator を使ってURLを発行します。

  1. 転送先URLを入力
    ユーザーが最終的にたどり着くページのURL。例: https://www.saison-technology.com/
  2. ページ識別名（slug）を入力
    リダイレクトURLの末尾IDになります。英数字・ハイフン・アンダースコアのみ。
    https://casa2024takayama.github.io/testLP/dev/ここに識別名が入る
    例: x_sns、nikochou_qr、summer_lp
  3. UTMパラメータを選択・入力
    utm_source（流入元）、utm_medium（カテゴリ）、utm_campaign（施策名）は必須。utm_content は任意。
  4. 管理タグ・ラベルを入力（任意）
    GA4 でフィルタするときに使うタグと、管理用のメモ。
  5. 「URL を生成」ボタンをクリック
    リダイレクトURL、QRコード、redirects.json 用エントリが一括生成されます。

4リダイレクトページの登録

UTM Generator で生成した JSON エントリを redirects.json に追記すると、GitHub Actions が自動でリダイレクトページを生成・デプロイします。

  1. UTM Generator の生成結果から JSON エントリを「コピー」ボタンでコピー
  2. GitHub の redirects.json を開き、鉛筆アイコン（Edit）をクリック
  3. 最後のエントリの } の後ろに カンマ , を追加し、コピーした JSON を貼り付け 
        // ← 既存の最後のエントリ
    },
    {
      "slug": "新しいエントリ",
      ...
    }
  ]
}
  4. 「Commit changes」で保存 → GitHub Actions が自動でビルド＆デプロイ（1〜2分）
  5. Actions タブで緑チェックが付いたら完了。リダイレクトURLにアクセスして動作確認。
⚠️ よくあるミス

カンマ忘れ: 既存エントリと新エントリの間にカンマ , がないと JSON が壊れてビルドが失敗します。
slug の重複: 同じ slug が既にある場合、上書きされます。別の識別名をつけてください。
Actions が赤バツ: redirects.json の JSON 構文エラーが原因のことが多いです。JSONLint で検証してください。

5動作確認

リダイレクトの確認

  1. 生成されたリダイレクトURLにアクセス（PCまたはスマホ）
  2. 0.8秒後に転送先ページに遷移すれば成功

GA4 リアルタイムでの確認

  1. GA4 を開き、左メニューの「リアルタイムの概要」をクリック
  2. 別のデバイス（スマホ等）でリダイレクトURLにアクセス
  3. 30秒以内に「イベント数：イベント名別」に link_click_redirect が表示されれば成功

6UTM 命名規則

パラメータ一覧

パラメータ役割記述ルール例
utm_source 流入元サイト 媒体名（プルダウンから選択 or 手入力） x, facebook, flyer

7FAQ

Q. 同じ slug を使いたい場合は？

slug は一意である必要があります。同じ slug で redirects.json に追記すると、後から追加した方で上書きされます。別の識別名（例: x_sns_v2）にしてください。

Q. GA4 のリアルタイムにイベントが出ない

リアルタイムレポートは過去30分のデータを表示します。アクセス直後に確認してください。また、広告ブロッカーが有効だと GA4 タグがブロックされる場合があります。

Q. GitHub Actions が失敗した（赤バツ）

redirects.json の JSON 構文エラーが最も多い原因です。カンマの過不足を確認するか、JSONLint に貼り付けて検証してください。

Q. 発行履歴が消えた

UTM Generator の発行履歴はブラウザの localStorage に保存されています。ブラウザのキャッシュクリアや別ブラウザでは表示されません。重要な履歴は CSV エクスポートで保存してください。

Q. 本番環境でも使える？

現在は GitHub Pages 上のテスト環境（/dev/ 配下）です。本番のリダイレクトページはマーケ部門が自社サイト上に構築します。UTM Generator と命名規則はそのまま流用できます。

Q. 既存の GA4 計測に影響はある？

ありません。リダイレクトページ経由のアクセスは link_click_redirect という専用イベントで記録されるため、既存の page_view やコンバージョン計測とは独立しています。
